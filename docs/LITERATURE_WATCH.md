# 文献自动更新说明

## 目标

本站的“定期自动更新”并不等于自动通读封闭数据库全文，而是：

- 定期检索公开可访问网页中的新增线索
- 把结果整理成“待审核公开线索”
- 自动写入站点数据文件
- 由站点前端展示最近线索，供人工继续核验

## 已实现内容

### 1. 公开线索配置

- 文件：`src/data/literature-watch-config.js`
- 作用：为张学文、刘祖贻、凃晋文分别配置检索词与优先来源域名

### 2. 自动更新脚本

- 文件：`scripts/update-literature.mjs`
- 命令：`npm run update:literature`
- 作用：
  - 检索公开网页结果
  - 抓取页面标题与摘要线索
  - 过滤已收录来源与明显低质量聚合结果
  - 写入 `src/data/literature-watch.generated.js`

### 3. 站内展示

- 首页展示最近 3 条公开线索
- `研究方法` 页展示最近 6 条公开线索
- 所有条目均标注为“自动检索公开线索 / 待审核”

### 4. 周期任务

- 文件：`.github/workflows/update-literature.yml`
- 触发：
  - 每周一北京时间上午 9 点左右自动运行
  - 也可在 GitHub Actions 手动触发

## 使用边界

- 自动结果仅代表“公开网页新增线索”，不代表已通读全文
- 自动结果不直接并入正式学术结论
- 自动结果不替代人工核验、来源甄别与内容提炼
- 如遇封闭数据库全文、同名作者歧义、转载重复等情况，必须人工判断

## 本地手动运行

```bash
npm run update:literature
```

运行后可检查：

- `src/data/literature-watch.generated.js`
- 首页“最新公开线索”
- `研究方法` 页“最新公开线索”
